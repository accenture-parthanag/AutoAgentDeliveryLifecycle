# A-ADLC React Implementation Guide

This document provides a production-ready React implementation framework for the Agentic Automation Development Lifecycle (A-ADLC) project, adhering to the McKinsey-inspired design system tokens.

## 1. Project Structure

We recommend a modular directory structure to maintain separation of concerns between shared UI components and phase-specific agent workspaces.

```text
src/
├── components/          # Shared UI (TopNavBar, SideNavBar, Footer)
├── design-system/       # Tokens, Tailwind config, and global styles
├── hooks/              # Custom logic for agent status & data fetching
├── views/              # Main screen components
│   ├── ExecutiveOverview.jsx
│   ├── RequirementHardening.jsx
│   ├── ArchitectureStrategy.jsx
│   ├── TechnicalDecomposition.jsx
│   └── BuildValidation.jsx
└── App.js               # Routing and Layout shell
```

## 2. Design System Integration (Tailwind Configuration)

To maintain the McKinsey aesthetic (Light theme, deep blue accents, serif headers), use the following configuration based on `{{DATA:DESIGN_SYSTEM:DESIGN_SYSTEM_1}}`.

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: '#051c2c', // McKinsey Blue
        surface: '#f9f9f9',
        'surface-container': '#ffffff',
        outline: '#dadada',
      },
      fontFamily: {
        serif: ['Source Serif 4', 'serif'], // For headers
        sans: ['Inter', 'sans-serif'],      // For body copy
      },
    },
  },
}
```

## 3. Core Layout Component (Shell)

The application uses a persistent layout with a `TopNavBar` and `SideNavBar`.

```jsx
import React from 'react';
import { TopNavBar, SideNavBar, Footer } from './components';

const AppLayout = ({ children }) => {
  return (
    <div className="min-h-screen bg-surface font-sans text-primary">

        <main className="flex-1 p-8 ml-64 mb-16">
          {children}
        </main>
      </div>
      <Footer />
    </div>
  );
};
```

## 5. Development Next Steps

1.  **Component Library:** Build out the `TopNavBar` and `SideNavBar` using the JSON definitions in your Components Inventory.
2.  **Navigation Flow:** Use `react-router-dom` to handle the transition between agent workspaces.
3.  **State Management:** Consider a global store (e.g., Context API or Zustand) to track the shared "Agent Status" across the lifecycle.
