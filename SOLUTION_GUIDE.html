Getting Started

AASDI Platform Solution Guide

Complete instructions for running and using the Agentic Automation Development Lifecycle Platform.

01 / Installation & Setup

Getting Started

Prerequisites

  • Node.js v14 or higher (recommend v16+)
  • npm v6 or higher (comes with Node.js)
  • A code editor (VS Code recommended)
  • Git for version control

Installation Steps

1

Clone or Download the Repository

Get the project files from your repository and navigate to the directory.

cd "path/to/Agent Development"
2

Install Dependencies

Install all required npm packages using npm.

npm install
3

Start the Application

Choose one of two startup modes below based on your needs.

02 / Running the Application

Startup Options

🚀 Easiest Way: Automated Startup Scripts

Use one of the provided startup scripts to automatically launch all three processes. Scripts automatically clean up any existing node processes before starting.

⭐ 1️⃣ PowerShell Script (Recommended)
.\start-all.ps1
✓ Opens 3 separate windows · ✓ Auto-cleanup old processes · ✓ Verifies startup
Stop with: .\stop-all.ps1
Arrange windows side-by-side for easy monitoring
2️⃣ Batch File Wrapper
start-all.bat
✓ Double-click from File Explorer · ✓ Auto-cleanup · ✓ Windows-native
Stop with: .\stop-all.bat
3️⃣ NPM Script (Simple)
npm run start:all
Runs all three in one terminal (mixed output)
4️⃣ Node.js Script (Cross-Platform)
node start-all.js
Works on Windows, Mac, and Linux

✅ Recommended Workflow: Use .\start-all.ps1 to start and .\stop-all.ps1 to stop. This provides a complete clean startup and shutdown with no leftover processes or windows.

Port Configuration

The startup script automatically sets the correct ports:

  • Frontend: PORT=3000 (React Dev Server)
  • Backend: PORT=5000 (Express API Server)
  • BA Agent: Listens for jobs from MongoDB queue

The frontend explicitly sets PORT=3000 to prevent port conflicts with the backend.

Organizing Your Windows

When the 3 windows open, arrange them side-by-side for easy monitoring:

  • Left window: Frontend (npm start on port 3000) - shows React dev server logs
  • Middle window: Backend (npm run server on port 5000) - shows API server logs
  • Right window: BA Agent (npm run ba-agent) - shows agent processing logs

Manual Startup: Three Separate Windows (If Scripts Don't Work)

Note: Use the startup scripts above instead. Only use this method if you encounter issues with the automated scripts.

Frontend Only (Development)

Start the React development server with hot reload. Access the UI at http://localhost:3000

npm start

✓ Fast startup · ✓ Hot module reload · ✗ Backend agents unavailable

Full Stack with Agents (Automated - Recommended)

Use one of the startup scripts above. They automatically manage all three processes for you.

If you prefer manual control, open three separate windows and follow the steps below.

Full Stack with Agents (Manual Setup)

✓ Complete Setup (3 Windows)
Window 1: Frontend
npm start
Opens http://localhost:3000 with hot reload
Window 2: Backend Server
npm run server
API server on http://localhost:5000 · Manages projects and jobs
Window 3: BA Agent Process
npm run ba-agent
Business Analyst agent using Claude CLI · Processes PDD analysis jobs

Step-by-step: Open three terminal windows in the project directory. Run each command above in its own window. Start them in order: Frontend → Backend → Agent. All three must be running for full functionality.

Simplified Setup (Frontend + Backend Only)

Use the npm run dev command to start both frontend and backend in one window (agent functionality unavailable).

npm run dev

✓ Easier startup · ✓ Basic functionality · ✗ No AI agent processing

Backend Only (Server)

Start only the backend server (useful for API testing or production).

npm run server

Server runs on http://localhost:5000

Build for Production

Create an optimized production build in the /build directory.

npm run build

💡 Key Point: The three-window setup is required for full functionality with AI agent processing. Each window runs a critical component that cannot be combined.

Quick Start Checklist (Using Startup Scripts)

Verifying Everything is Running

  • Frontend: http://localhost:3000 shows the dashboard
  • Backend: http://localhost:5000/api/health returns {"status":"ok"}
  • BA Agent: Console shows "✓ BA Agent running" and listens for jobs

Stopping the Services

When you're done, stop all services using one of these methods:

⭐ Option 1: Stop Script (Recommended)
.\stop-all.ps1
✓ Terminates all node processes
✓ Closes spawned PowerShell windows
✓ Frees ports 3000 and 5000
✓ Complete clean shutdown
Option 2: Batch File
.\stop-all.bat
Windows batch alternative - does the same as PowerShell version
Option 3: Close Windows Manually
Click the X button on each of the three service windows
Option 4: Keyboard Shortcut
Press Ctrl+C in each service window to stop individually

💡 Tip: Use the stop script for a complete clean shutdown that also closes the PowerShell windows spawned by the startup script. This prepares the system for a fresh restart.

03 / Key Functionalities

Platform Features Overview

The AASDI Platform implements the Agentic Automation Development Lifecycle framework with five specialized agent roles and an intuitive workflow system.

📊 Project Dashboard

View all projects, track progress metrics, monitor outstanding requests across workflow stages, and see recent project activity at a glance.

📋 Project Management

Create new projects, view detailed project information, track current phase and target dates, and manage project lifecycle.

🤖 Five Agent Roles

Business Analyst, Architect, Tech Lead, Developer, and QA Agent — each with specialized capabilities for different workflow stages.

📄 PDD Workflow

Complete Product Design Document workflow including document upload, BA analysis, gap identification, and approval process.

05 / Agent Capabilities

Five Specialized Agents

1. Business Analyst Agent

Analyzes Product Design Documents and provides business-level insights.

  • Reviews PDD completeness and clarity
  • Identifies missing business requirements
  • Generates gap analysis reports
  • Validates success criteria and scope

2. Architect Agent

Designs high-level solution architecture and system design.

  • Creates solution design documents (SDD)
  • Designs system architecture
  • Specifies component interactions
  • Defines integration points and interfaces

3. Tech Lead Agent

Decomposes solutions into technical components and specifications.

  • Creates technical design documents (TDD)
  • Defines low-level system components
  • Specifies technical requirements
  • Plans implementation approach

4. Developer Agent

Generates and implements code based on specifications.

  • Generates source code
  • Implements features and components
  • Follows coding standards and best practices
  • Creates implementation documentation

5. QA Agent

Manages testing and quality assurance throughout the lifecycle.

  • Plans testing strategy (SIT and UAT)
  • Creates test cases and scenarios
  • Validates functionality and performance
  • Reports defects and quality metrics

09 / Database Management

Clear Database & Start Fresh

To reset the database and start with a clean slate:

node clear-db.js

What it does:

  • Connects to MongoDB (aadlc database)
  • Drops all collections (projects, jobs, etc.)
  • Prepares database for fresh testing

Use this when:

  • Starting a fresh test cycle
  • Testing the complete PDD workflow again
  • Cleaning up old projects and submissions

💾 Before clearing: Make sure you've stopped the application with .\stop-all.ps1, otherwise MongoDB operations may be locked.

10 / Troubleshooting

Common Issues & Solutions

Port Already in Use

Problem: "Port 3000 is already in use"

Solution: Kill the existing process or specify a different port:

PORT=3001 npm start

Dependencies Not Installing

Problem: npm install fails with errors

Solution: Clear npm cache and reinstall:

npm cache clean --force
rm -rf node_modules package-lock.json
npm install

Backend Connection Issues

Problem: "Cannot connect to backend" or 404 errors on API calls

Solution: Ensure backend is running with npm run dev or npm run server

Styling Not Applied

Problem: CSS changes not showing

Solution: Hard refresh browser (Ctrl+Shift+R) to clear cached CSS

MongoDB Connection Failed

Problem: Backend fails to start with MongoDB errors

Solution: Check MongoDB is running and connection string is correct in server/index.js

Agent Not Processing Jobs

Problem: Created a project but BA Agent analysis doesn't start

Solution: Ensure all three windows are running. The BA Agent window must show "✓ BA Agent running" in the console. Check that npm run ba-agent started without errors.

Forgot to Open One of the Three Windows

Problem: Application partially works but some features unavailable

Solution: Open the missing window and start the corresponding process:

  • Missing frontend? Start npm start
  • Missing backend? Start npm run server
  • Missing agent? Start npm run ba-agent

Can't Stop Services or Ports Still in Use

Problem: Ctrl+C isn't working or ports 3000/5000 show "address already in use"

Solution: Use the stop script to forcefully terminate all processes:

.\stop-all.ps1

Or kill processes manually:

taskkill /IM node.exe /F
